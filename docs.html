🔥 What Was Hard

  • Deprecated value field in Cloudflare provider v4 Terraform validated fine but threw deprecation warnings on every field. The provider silently renamed value → content in v4. ✓ Fixed: replaced all value = with content = in main.tf
  • Variable name mismatch for TF_VAR_ env injection The .env file exports TF_VAR_cloudflare_zone_id but the Terraform variable was named zone_id, so Terraform ignored the env var and prompted interactively. ✓ Fixed: renamed variable to cloudflare_zone_id to match the env key
  • cloudfire records already existed in Cloudflare (proxied=true) Terraform tried to create records that were already there and failed with "already exists". They must have been added manually before — Terraform had no state for them. ✓ Fixed: terraform import cloudflare_record.cloudfire_cname <zone>/<id> to adopt them into state
  • cloudfire SSL broke after turning off Cloudflare proxy The record was previously proxied=true — Cloudflare was terminating TLS. Switching to DNS-only exposed that Azure had no managed cert for cloudfire.mysak.fun (SSL state was Disabled). ✓ Fixed: az webapp config ssl create + ssl bind --ssl-type SNI to provision and attach a free managed cert
  • asuid TXT record — same value for all services Azure App Service and Container Apps use a per-subscription verification ID (not per-app), so asuid.penny and asuid.cloudfire share the exact same TXT value. Easy to miss if you expect per-app tokens. ✓ Verified: same hash 24C4FD8D…A540A works for both
  • penny cold-start false-alarm HTTPS to penny.mysak.fun timed out during testing. Looked like a DNS or SSL problem but was actually the Container App scaling from 0 replicas on first request (minReplicas: null). ✓ Expected behaviour — set minReplicas=1 if cold starts are unacceptable
